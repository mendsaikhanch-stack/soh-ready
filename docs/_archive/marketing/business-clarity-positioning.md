# ХОТОЛ Business Clarity & Positioning

**Зорилго:** ХОТОЛ-ын бизнесийг шинжилж, хэнд чиглэх, ямар асуудал шийдэх, юугаараа ялгардаг, хүмүүс үнэ цэнийг минь шууд ойлгож чадахаар хэрхэн байр суурилуулах ёстойг тодорхой тогтоох.

> Энэ нь өмнөх бүх документуудыг **нэг нүүрэн дээр нэгтгэх "founding document"** юм.

---

## 📦 1. Бизнесийн товч (Offer)

**Бүтээгдэхүүн:** ХОТОЛ — Монголын СӨХ-нд зориулсан комплекс менежмент апп.

**Багц 4 цөм функц:**

| Модуль | Юу хийдэг |
|--------|-----------|
| 💳 **Төлбөр** | СӨХ-ний сар тутмын төлбөр, нэхэмжлэх, баримт, сануулга |
| 📊 **Тайлан** | Орлого/зарлагын real-time ил тод тайлан гишүүн бүрд |
| 🗳️ **Цуглаан** | Онлайн санал хураалт, протокол, ирц бүртгэл |
| 🔔 **Харилцаа** | Мэдэгдэл, гомдол, цуглаан, мэдээллийн сувгийг нэгтгэх |

**Үнийн загвар:** Хороолол тус бүр сар тутмын subscription (айлын тоонд тулгуурлан). Эхний 3 сар үнэгүй (first 100 СӨХ).

## 🎯 2. ХЭН-Г ТАРГЕТ ХИЙХ ВЭ (Target Audience)

### Цөмийн зорилтот сегмент: **B2B2C dual persona**

```
┌─────────────────────────────────────────────────────────┐
│  PRIMARY BUYER:  СӨХ-ний тэргүүн (45-70 нас)            │
│                  → Гэрээ хийдэг, мөнгө төлдөг            │
│                                                          │
│  CO-INFLUENCER:  Оршин суугчид (25-50 нас)              │
│                  → Тэргүүндээ "ХОТОЛ авах" гэж шахдаг   │
└─────────────────────────────────────────────────────────┘
```

### ICP (Ideal Customer Profile) — нарийвчилсан

| Шинж чанар | Тодорхойлолт |
|------------|--------------|
| **Хорооллын хэмжээ** | 50-300 айл (sweet spot 100-150) |
| **Байршил** | Эхэндээ УБ хот — Хан-Уул, СБД, ЧД (дунд анги) |
| **СӨХ-ний нас** | 3+ жил үйл ажиллагаатай (chaos шатнаас гарсан) |
| **Тэргүүний хувийн шинж** | Технологид нээлттэй, дахин сонгогдох хүсэлтэй |
| **Сар бүрийн төсөв** | ₮5+ сая (subscription төлөх боломжтой) |
| **Painted-in pain** | Сүүлийн 12 сард дор хаяж 1 мэдээ хямрал өнгөрсөн (төлбөрийн маргаан, шилжилт, аудит) |

### Маркетийн ангилал — **Beachhead → Expansion**

```
Жил 1: УБ — дунд оронгийн орон сууцны хороолол (300+ хороолол)
Жил 2: УБ — luxury + low-tier
Жил 3: Орон нутгийн (Эрдэнэт, Дархан) + ажлын байр management
Жил 5: Олон улсын (ОУ-ын СӨХ-ний дижитал шилжилт)
```

## ❓ 3. ЯМАР АСУУДАЛ ШИЙДДЭГ ВЭ (Core Problem)

### Нэг өгүүлбэрээр:

> **"Монголын СӨХ-нүүд цаасан баримт, Excel, Messenger group-аар ажилладаг — энэ нь тэргүүний цаг алддаг, оршин суугчдын итгэлийг хорлодог, орон сууцны үнэ цэнг бууруулдаг."**

### 4 түвшний асуудал:

| Түвшин | Асуудал | ХОТОЛ-ын шийдэл |
|--------|---------|------------------|
| **Тэргүүн** | Цаг алддаг, шүүмжлэгддэг, хууль зөрчиж магадгүй | Auto-process + ил тод тайлан + хуулийн compliance |
| **Оршин суугч** | Мөнгөо хаашаа явсныг мэдэхгүй, дуу хоолой алга | Real-time транспарент тайлан + онлайн санал |
| **Хороолол** | Менежмент муу → орон сууцны үнэ буурдаг | Дижитал захиргаа → community trust → property value |
| **Систем** | Монгол улсын СӨХ-ний дата хаана ч цуглуулагдаагүй | Бенч-марк дата → бодлогын нөлөөлөл |

## 🌟 4. ЮУГААРАА ЯЛГАГДДАГ ВЭ (Differentiation)

### Бусдаас ялгарах **3 sustainable moat**:

#### 🛡️ MOAT 1: "Систем", "тоол" биш — **"Дижитал захиргаа"**

> Бусад: payment tool / Excel-ийн орлуулагч
> ХОТОЛ: бүхэлдээ хорооллын дижитал захиргаа (governance layer)

#### 🛡️ MOAT 2: **Тэргүүн нарын карьер шинжтэй** community

> Бусад: бизнесийн харилцагч (vendor)
> ХОТОЛ: тэргүүний "академи" + сертификат + дахин сонгогдох хүчин зүйл

#### 🛡️ MOAT 3: **Транспарент business** model

> Бусад: hidden pricing, "freemium trap"
> ХОТОЛ: жил бүр open report. "Манай орлого ₮Х, цалин ₮Y, server ₮Z."

### Эдгээрийн нэг хослуулсан давуу тал:

```
Network effects (хороолол олшроход дата хүчтэй болно)
        ×
Community switch-cost (тэргүүн солигдсон ч academy үлдэнэ)
        ×
Trust premium (transparent business → pricing power)
        =
Compounding moat
```
